## 2. WP-024 Fix: `cleanup_nodes` Multi-ID Support

**Current signature** (`tests/conftest.py` line 78):
```python
def cleanup_nodes(driver, *memory_ids, extra_ids: dict | None = None) -> None:
```
Current loop `for label, nid in (extra_ids or {}).items()` assumes each label maps to a single `str` id.

**Required change:** Accept `dict[str, str | list[str]]` â€” normalize to list internally:

```python
def cleanup_nodes(driver, *memory_ids, extra_ids: dict[str, str | list[str]] | None = None) -> None:
    with driver.session() as session:
        for mid in memory_ids:
            session.run("MATCH (m:Memory {id: $id}) DETACH DELETE m", id=mid)
        for label, nid in (extra_ids or {}).items():
            ids = [nid] if isinstance(nid, str) else nid
            for single_id in ids:
                session.run(f"MATCH (n:{label} {{id: $id}}) DETACH DELETE n", id=single_id)
```

This is backward-compatible â€” all existing callers pass `{label: single_str}` and still work.

## 5. Test Infrastructure Notes

### `knowledge_client` fixture
Integration tests need `ENABLE_KNOWLEDGE_LAYER=true` at app construction time. Create a session-scoped `knowledge_client` fixture in `test_wp076_integration.py`:

```python
@pytest.fixture(scope="module")
def knowledge_client(test_driver):
    os.environ["ENABLE_KNOWLEDGE_LAYER"] = "true"
    import importlib
    import memory_service.config as cfg_mod
    import memory_service.main as main_mod
    importlib.reload(cfg_mod)
    importlib.reload(main_mod)
    main_mod.app.state.driver = test_driver
    with TestClient(main_mod.app) as c:
        yield c
```

### Seeding strategy
- **Knowledge nodes** (Framework, Control, Document, Chunk, Norm): seed via HTTP endpoints to test the full stack
- **Graph-model-only nodes** (Precept, BusinessAttribute, Organisation, Jurisdiction, ADDRESSES, FULFILS, REQUIRES, APPLIES_IN edges): seed via direct Cypher since no HTTP endpoints exist for these
- **Memory nodes**: seed via `POST /memory`
- **Cross-layer edges** (ABOUT_CONTROL, CITES_DOC): created via `POST /memory` with `control_ids`/`doc_ids`

### Cleanup strategy
Each test class uses an `autouse` teardown fixture that deletes by ID prefix:
```cypher
MATCH (n) WHERE n.id STARTS WITH $prefix DETACH DELETE n
```
This catches any nodes created by test failures (not just the happy path).

### Import audit test
Include one static analysis test in `test_wp076_separation.py`:
```python
def test_import_audit_knowledge_bridge_is_sole_cross_layer_importer():
    """knowledge_bridge.py must be the only module importing from both layers."""
    import ast, pathlib
    src = pathlib.Path("memory_service")
    for py_file in src.glob("*.py"):
        if py_file.name == "knowledge_bridge.py":
            continue
        tree = ast.parse(py_file.read_text())
        imports = [n for n in ast.walk(tree) if isinstance(n, (ast.Import, ast.ImportFrom))]
        names = [getattr(n, "module", "") or "" for n in imports]
        has_memory = any("memory_repo" in n for n in names)
        has_knowledge = any("knowledge_repo" in n for n in names)
        assert not (has_memory and has_knowledge), (
            f"{py_file.name} imports from both memory_repo and knowledge_repo â€” "
            "only knowledge_bridge.py is permitted to cross the layer boundary"
        )
```
