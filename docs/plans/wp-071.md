# WP-071 Plan: Knowledge Layer Search API

## Summary

Adds read/search endpoints to the existing `knowledge_routes.py` and `knowledge_repo.py` (created in WP-070). Delivers vector search over Controls and Chunks, catalogue listing for Norms and Documents, and a diagnostic endpoint for jurisdiction coverage gaps. All endpoints are guarded by the `enable_knowledge_layer` feature flag. No `recall_count` increment ‚Äî knowledge search is reference lookup, not episodic recall. All search endpoints function with zero Memory nodes in the graph (ADR-001 dual-mode requirement).

## Pydantic models (add to `knowledge_routes.py`)

```python
# --- Search request models ---

class ControlSearchRequest(BaseModel):
    query: str
    limit: int = 10
    framework_id: Optional[str] = None   # filter: only Controls from this Framework

class ChunkSearchRequest(BaseModel):
    query: str
    limit: int = 10
    doc_id: Optional[str] = None         # filter: only Chunks from this Document

# --- Search result models ---

class ControlHit(BaseModel):
    id: str
    name: str
    description: Optional[str] = None
    framework_id: str
    created_at: str
    distance: float

class ChunkHit(BaseModel):
    id: str
    text: str
    sequence: int
    doc_id: str
    created_at: str
    distance: float
```

No new response models are needed for `list_norms` and `list_documents` ‚Äî they reuse `NormResponse` and `DocumentResponse` (already defined in WP-070). `list_incomplete_jurisdictions` returns a simple JSON dict (see repo function below).

## Function signatures (`knowledge_repo.py`)

```python
def search_controls(
    session,
    query_embedding: list[float],
    limit: int,
    framework_id: str | None,
) -> list[dict]:
    """Vector search over ctrl_embedding_idx.

    Returns list of dicts: {id, name, description, framework_id, created_at, distance}.
    No recall_count increment ‚Äî knowledge search is reference lookup.
    """

def search_chunks(
    session,
    query_embedding: list[float],
    limit: int,
    doc_id: str | None,
) -> list[dict]:
    """Vector search over chunk_embedding_idx.

    Returns list of dicts: {id, text, sequence, doc_id, created_at, distance}.
    No recall_count increment.
    """

def list_norms(session) -> list[dict]:
    """Return all Norm nodes, ordered by name.

    Returns list of dicts: {id, name, text, status, effective_date, created_at}.
    """

def list_documents(session) -> list[dict]:
    """Return all Document nodes, ordered by title.

    Returns list of dicts: {id, title, doc_type, source_url, created_at}.
    """

def list_incomplete_jurisdictions(session) -> dict:
    """Return Norms and Controls with no APPLIES_IN edges.

    Returns dict:
      {
        "norms_without_jurisdiction": [{"id": ..., "name": ...}, ...],
        "controls_without_jurisdiction": [{"id": ..., "name": ...}, ...]
      }
    Useful for data quality / gap diagnostic.
    """
```

## Cypher query designs

### `search_controls`

```cypher
CALL vector_search.search("ctrl_embedding_idx", $limit, $query_vec)
YIELD node AS c, distance
WITH c, distance
WHERE ($framework_id IS NULL OR c.framework_id = $framework_id)
RETURN c.id AS id, c.name AS name, c.description AS description,
       c.framework_id AS framework_id, c.created_at AS created_at,
       distance
ORDER BY distance ASC
```

Parameters: `limit`, `query_vec`, `framework_id` (nullable).

### `search_chunks`

```cypher
CALL vector_search.search("chunk_embedding_idx", $limit, $query_vec)
YIELD node AS ch, distance
WITH ch, distance
WHERE ($doc_id IS NULL OR ch.doc_id = $doc_id)
RETURN ch.id AS id, ch.text AS text, ch.sequence AS sequence,
       ch.doc_id AS doc_id, ch.created_at AS created_at,
       distance
ORDER BY distance ASC
```

Parameters: `limit`, `query_vec`, `doc_id` (nullable).

### `list_norms`

```cypher
MATCH (n:Norm)
RETURN n.id AS id, n.name AS name, n.text AS text, n.status AS status,
       n.effective_date AS effective_date, n.created_at AS created_at
ORDER BY n.name ASC
```

### `list_documents`

```cypher
MATCH (d:Document)
RETURN d.id AS id, d.title AS title, d.doc_type AS doc_type,
       d.source_url AS source_url, d.created_at AS created_at
ORDER BY d.title ASC
```

### `list_incomplete_jurisdictions`

```cypher
-- Norms without APPLIES_IN
MATCH (n:Norm)
WHERE NOT (n)-[:APPLIES_IN]->(:Jurisdiction)
RETURN n.id AS id, n.name AS name
ORDER BY n.name ASC

-- Controls without APPLIES_IN
MATCH (c:Control)
WHERE NOT (c)-[:APPLIES_IN]->(:Jurisdiction)
RETURN c.id AS id, c.name AS name
ORDER BY c.name ASC
```

Run as two separate `session.run()` calls; combine results into the dict.

## Route handlers (add to `knowledge_routes.py`)

```python
@router.post("/search/controls", response_model=List[ControlHit])
async def search_controls(req: ControlSearchRequest, request: Request) -> List[ControlHit]:
    query_vec = get_embedding(req.query, model_name=settings.knowledge_embedding_model)
    with request.app.state.driver.session() as session:
        hits = knowledge_repo.search_controls(session, query_vec, req.limit, req.framework_id)
    return [ControlHit(**h) for h in hits]


@router.post("/search/chunks", response_model=List[ChunkHit])
async def search_chunks(req: ChunkSearchRequest, request: Request) -> List[ChunkHit]:
    query_vec = get_embedding(req.query, model_name=settings.knowledge_embedding_model)
    with request.app.state.driver.session() as session:
        hits = knowledge_repo.search_chunks(session, query_vec, req.limit, req.doc_id)
    return [ChunkHit(**h) for h in hits]


@router.get("/norms", response_model=List[NormResponse])
async def list_norms(request: Request) -> List[NormResponse]:
    with request.app.state.driver.session() as session:
        norms = knowledge_repo.list_norms(session)
    return [NormResponse(**n) for n in norms]


@router.get("/documents", response_model=List[DocumentResponse])
async def list_documents(request: Request) -> List[DocumentResponse]:
    with request.app.state.driver.session() as session:
        docs = knowledge_repo.list_documents(session)
    return [DocumentResponse(**d) for d in docs]


@router.get("/incomplete-jurisdictions")
async def list_incomplete_jurisdictions(request: Request) -> dict:
    with request.app.state.driver.session() as session:
        result = knowledge_repo.list_incomplete_jurisdictions(session)
    return result
```

Note: `List` import already present in `main.py`; add `from typing import List` to `knowledge_routes.py` imports.

## Risks and open questions

| ID | Risk | Mitigation |
|----|------|-----------|
| R1 | `vector_search.search()` returns up to `$limit` nodes before the `WHERE` filter is applied (same as memory search) | Known behaviour; documented in the `_SEARCH_QUERY_TEMPLATE` comment in memory_repo.py. Document same caveat in knowledge_repo.py |
| R2 | `list_incomplete_jurisdictions` requires `APPLIES_IN` edges to exist ‚Äî but `Jurisdiction` nodes are not part of WP-070 or WP-071 | The query returns all Norms/Controls when no Jurisdictions exist (i.e., all are "incomplete"). This is correct for the diagnostic purpose. |
| R3 | `GET /knowledge/norms` and `GET /knowledge/documents` have no pagination | Acceptable for v1; data volumes are small at this stage. Add pagination in a later WP if needed. |
